🧠 gameState.js

C'est le "cerveau". Il stocke toutes les rooms en mémoire RAM (pas en BDD). Chaque room contient la liste des joueurs, la syllabe en cours, le timer, les mots déjà utilisés.

⚡ gamesocket.js

C'est le "chef d'orchestre". Il reçoit les actions des joueurs, valide les mots, fait exploser la bombe, élimine les joueurs, détecte la fin de partie.

03 — Déroulement

Flux du jeu

1

Création de la room

Le host entre son pseudo et clique "Créer". Le serveur génère un code à 6 caractères unique (ex: AB3C7F). Les autres rejoignent avec ce code.

2

Lancement (host uniquement)

Le host clique "Lancer". Minimum 2 joueurs requis, maximum 20. Le serveur démarre le premier tour.

3

Tour de jeu

Un joueur reçoit une syllabe (ex: OUR). Il doit taper un mot qui la contient avant que la bombe explose. Le timer est secret et aléatoire.

4

Validation du mot

Le serveur vérifie : le mot contient la syllabe ? Il est dans le dictionnaire (336k mots) ? Il n'a pas déjà été utilisé cette partie ? Si tout OK → +points, tour suivant.

5

Explosion

Si le temps expire : -1 vie. À 0 vies, le joueur est éliminé. La partie continue jusqu'au dernier survivant.

6

Fin de partie

Le dernier joueur vivant gagne. Les scores sont sauvegardés en BDD. Le classement final s'affiche.

S → C
05 — Mécanique clé

Timer secret

Le timer n'est jamais envoyé au client. Seul le serveur sait quand la bombe explose. La bombe tremble de façon aléatoire — impossible de deviner le temps restant.

Réduction progressive

Chaque mot trouvé réduit le plafond du timer de 1 seconde. La pression monte sans que les joueurs s'en rendent compte.

function dureeAleatoire(reductionMs = 0) {
  const max = Math.max(8000, 45000 - reductionMs)
  return Math.floor(Math.random() * (max - 8000 + 1)) + 8000
}

// À chaque mot trouvé, dans gamesocket.js :

07 — Démarrage

Lancer le projet

1

Installer les dépendances

$cd TicTacBoom/backend && npm install
2

Démarrer le serveur

$npm start

→ Le serveur tourne sur http://localhost:3001

3

Test local (2 onglets)

Ouvrir localhost:3001 dans deux onglets. Un crée, l'autre rejoint.

4

Exposer sur le réseau (ngrok)

$ngrok http 3001

→ Partager l'URL ngrok à la classe. Tout le monde se connecte dessus.
